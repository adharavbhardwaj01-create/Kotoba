**Date**: July 13, 2026  
**Purpose**: Identify free resources for Japanese learning app

## 1. Jisho.org API

**Name**: Jisho.org API  
**URL**: https://jisho.org/api/v1/search/words?keyword={query}  
**Free Tier**: Completely free, no API key required  
**Rate Limits**: Not officially documented; community reports suggest ~100 requests/minute is safe  
**Data Format**: JSON  
**License**: Uses JMdict/Kanjidic2/JMnedict data from EDRDG (used in conformance with their license)  
**Restrictions**: No official API documentation; unofficial/undocumented API  
**Endpoints**:
- Word search: `/api/v1/search/words?keyword={word}`
- Kanji search: `/api/v1/search/kanji?keyword={kanji}`
- Sentence search: `/api/v1/search/sentences?keyword={sentence}`

**Recommendation**: Best for dictionary lookups. Well-structured JSON responses with readings, definitions, JLPT levels, and parts of speech. Consider caching results locally to reduce API calls.

## 2. JMdict/EDICT Dictionary Data

**Name**: JMdict (Japanese-Multilingual Dictionary)  
**URL**: https://www.edrdg.org/jmdict/j_jmdict.html  
**Free Tier**: Free to download and use  
**Data Format**: Original XML; simplified JSON available via jmdict-simplified  
**License**: EDRDG License (free for non-commercial use with attribution)  
**Restrictions**: Must include attribution to EDRDG; data updated periodically  
**JSON Conversion**: https://github.com/scriptin/jmdict-simplified (379 stars, actively maintained)

**Features**:
- 200,000+ entries
- Multiple language translations (English, German, Russian, etc.)
- JLPT level tags
- Parts of speech
- Example sentences (JMdict_e version)

**Recommendation**: Excellent for offline dictionary. Download the "common-only" JSON version for smaller size. Integrate into app database for fast lookups without API dependency.

## 4. Google Translate API

**Name**: Google Cloud Translation API  
**URL**: https://cloud.google.com/translate  
**Free Tier**: 
- $300 free credit for new accounts (90 days)
- No permanent free tier for Translation API specifically
- Rate limit: 6M characters/minute (general model), 300K requests/minute (v2)

**Pricing** (after free trial):
- Basic (v2): $20 per 1M characters
- Advanced (v3): $20 per 1M characters
- Custom models: $20 per 1M characters

**Restrictions**:
- Requires Google Cloud account with billing enabled
- API key or service account required
- Usage tracked and billed after free credit exhausted

**Recommendation**: Not suitable for long-term free usage. Consider only for development/testing or if budget allows.

## 5. DeepL API

**Name**: DeepL API  
**URL**: https://www.deepl.com/pro-api  
**Free Tier**:
- Free plan available with limited characters
- Free API endpoint: `https://api-free.deepl.com/v2/translate`
- Exact free character limit varies by region (typically 500,000 characters/month)

**Rate Limits**:
- Free plan: 5 translations/second
- Pro plan: Higher limits

**Data Format**: JSON  
**License**: Commercial license required for production use  
**Restrictions**:
- Free tier for evaluation only
- Commercial use requires paid plan
- Monthly character limits

**Recommendation**: Good quality translation but limited free tier. Better for premium features or if budget allows.

## 6. Open-Source Translation Alternatives

### 6.1 LibreTranslate
**Name**: LibreTranslate  
**URL**: https://github.com/LibreTranslate/LibreTranslate  
**Stars**: 15.4k  
**License**: AGPL-3.0  
**Features**:
- Self-hosted, offline capable
- No API key required when self-hosted
- REST API compatible
- Supports 30+ languages including Japanese

**Requirements**:
- Python 3.7+
- GPU recommended for good performance
- Storage for language models (~1-2GB per language pair)

**Recommendation**: Best open-source option. Self-host for unlimited free translations. Consider for privacy-focused users.

### 6.2 Argos Translate
**Name**: Argos Translate  
**URL**: https://github.com/argosopentech/argos-translate  
**Stars**: 6.3k  
**License**: MIT / CC0  
**Features**:
- Offline translation library
- Python library, CLI, and GUI
- Uses OpenNMT for translations
- Language model packages (.argosmodel)

**Requirements**:
- Python 3.6+
- ~500MB-2GB storage per language pair

**Recommendation**: Good as underlying engine for LibreTranslate. Can be integrated directly into app for offline translation.

## Summary & Recommendations

### For Kotoba App:

| Resource | Use Case | Cost | Notes |
|----------|----------|------|-------|
| JMdict (local) | Dictionary lookups | Free | Download JSON, store locally |
| Jisho API | Fallback dictionary | Free | Cache results, respect rate limits |
| LibreTranslate | Translation | Free (self-host) | Best for privacy, offline use |
| Argos Translate | Offline translation | Free | Integrate directly for offline mode |
| DeepL API | Premium translation | Limited free | Consider for Pro features |
| Google Translate | Backup translation | Paid | Avoid for free tier |

### Implementation Strategy:

1. **Primary Dictionary**: Use JMdict JSON locally (fast, offline, comprehensive)
2. **Online Fallback**: Jisho API with caching (respects rate limits)
3. **Translation**: LibreTranslate self-hosted or Argos Translate embedded
4. **Premium Feature**: DeepL API for enhanced translation quality
5. **Avoid**: Google Translate API (costly for production use)

### License Compliance:

- JMdict: Include EDRDG attribution in app
- KanjiVG: Include CC-BY-SA 3.0 attribution
- Tatoeba: Include CC-BY 2.0 attribution
- LibreTranslate: AGPL-3.0 requires source code disclosure if distributed
- Argos Translate: MIT/CC0 - most permissive
