## 1. KANJIDIC2 - Comprehensive Kanji Dictionary

- **Name**: KANJIDIC2
- **URL**: https://www.edrdg.org/ (via EDRDG)
- **Data Format**: XML (kanjidic2.xml)
- **License**: Creative Commons Attribution-ShareAlike License v4.0
- **Completeness**: ~13,000+ kanji entries including JIS Level 1-4 kanji
- **Content**:
  - Kanji readings (on'yomi, kun'yomi)
  - English meanings
  - Stroke count
  - Radical information
  - JLPT levels
  - Grade levels (Jouyou kanji)
  - Unicode code points
  - Chinese readings
  - Korean readings
- **How to Integrate**:
  - Direct XML parsing from official EDRDG source
  - Use pre-converted JSON from `scriptin/jmdict-simplified` (GitHub, 379 stars)
  - Libraries: `neocl/jamdict` (Python), `gojp/kanjidic2` (Go)
- **Notes**: Daily updates from source database. The JSON conversion by scriptin is recommended for easier parsing.

## 2. Kanji Stroke Order Data

- **Name**: KanjiVG (Primary), Kanji Stroke Order
- **URL**: https://github.com/KanjiVG/kanjivg
- **Data Format**: SVG (individual files per kanji) + XML index
- **License**: Creative Commons Attribution-ShareAlike 3.0
- **Completeness**: 10,000+ kanji with complete stroke order data
- **Content**:
  - Individual SVG files for each kanji
  - Stroke order paths
  - Stroke types (horizontal, vertical, curve, etc.)
  - Radical information
  - Component breakdown
- **How to Integrate**:
  - Clone the KanjiVG repository
  - Use the `kvg-lookup.py` utility for querying
  - Process SVG files to extract stroke paths
  - Build stroke order animation from path data
- **Notes**: Last release: r20250816. Includes both regular and variant forms.

## 3. JMdict - Japanese-Multilingual Dictionary

- **Name**: JMdict/EDICT
- **URL**: https://www.edrdg.org/jmdict/j_jmdict.html
- **Data Format**: XML (JMdict.xml, JMdict_e.xml)
- **License**: EDRDG License (free for commercial/non-commercial use with attribution)
- **Completeness**: 200,000+ entries
- **Content**:
  - Japanese words and readings
  - Kanji representations
  - English translations (and 10+ other languages)
  - Part of speech
  - Example sentences
  - Common word indicators
  - JLPT levels (derived from tags)
- **Does it include kanji readings?**: YES
  - JMdict contains kanji readings for vocabulary entries
  - Readings are marked as common (ichi1, news1, spec1/2, gai1)
  - For kanji-specific readings (not vocabulary), use KANJIDIC2
- **How to Integrate**:
  - Use `scriptin/jmdict-simplified` for JSON format
  - Common-only version available for smaller dataset
  - Language-specific versions available (English, German, etc.)
- **Notes**: Updated daily from source database.

## 4. KanjiVG - Stroke Order SVG Data

- **Name**: KanjiVG (Kanji Vector Graphics)
- **URL**: https://kanjivg.tagaini.net/
- **GitHub**: https://github.com/KanjiVG/kanjivg
- **Data Format**: SVG files, XML index, JSON index
- **License**: Creative Commons Attribution-ShareAlike 3.0
- **Completeness**: 10,000+ kanji
- **Release Files**:
  - `kanjivg-YYYYMMDD-main.zip` - Non-variant SVG files
  - `kanjivg-YYYYMMDD-all.zip` - All variants
  - `kanjivg-YYYYMMDD-stripped.zip` - Without custom attributes
  - `kanjivg-YYYYMMDD.xml.gz` - Legacy single XML format
- **How to Integrate**:
  - Download main release for standard kanji
  - Parse SVG path data for stroke animation
  - Use `kvg-index.json` for quick lookups
  - Build stroke order visualization from path elements
- **Notes**: Well-maintained project with Python utilities included.

## 5. Free Kanji Lookup APIs

### 5.1 Jisho API
- **URL**: https://jisho.org/api/v1/search/words
- **Format**: JSON REST API
- **License**: Based on JMdict (EDRDG License)
- **Features**:
  - Word search with kanji/kana
  - Reading lookup
  - English definitions
  - JLPT levels
  - Common word indicators
- **Example**: `https://jisho.org/api/v1/search/words?keyword=漢字`
- **Rate Limits**: Not documented (use responsibly)

### 5.2 WWWJDIC
- **URL**: https://www.edrdg.org/cgi-bin/wwwjdic/wwwjdic
- **Format**: Various interfaces (web, CGI)
- **License**: EDRDG License
- **Features**:
  - Multi-radical kanji lookup
  - Hand-written kanji input
  - Four-corner method
  - Extensive dictionary collection

### 5.3 KanjiVG API (Community)
- **URL**: Various community-built APIs using KanjiVG data
- **Format**: REST/GraphQL
- **Notes**: No official API, but multiple open-source implementations exist

## Integration Recommendations for Kotoba

### Data Pipeline Architecture
```
1. Download Sources
   ├── KANJIDIC2 (kanjidic2.xml) → kanji data
   ├── JMdict (JMdict_e.xml) → vocabulary data  
   ├── KanjiVG → stroke order SVGs
   └── KRADFILE → radical decomposition

2. Convert to JSON (use scriptin/jmdict-simplified)
   ├── kanjidic.json (from KANJIDIC2)
   ├── jmdict.json (from JMdict)
   ├── kanjivg.json (from KanjiVG index)
   └── kradfile.json (from KRADFILE)

3. Store in App Database
   ├── SQLite (local app data)
   └── IndexedDB (web app)

4. Build Features
   ├── Kanji lookup → KANJIDIC2 + JMdict
   ├── Stroke order animation → KanjiVG
   ├── Flashcard generation → All sources
   ├── Radical search → KRADFILE + KanjiVG
   └── Vocabulary → JMdict
```

### Priority Order for Implementation
1. **KANJIDIC2 + JMdict** (JSON from scriptin) - Core kanji/vocabulary data
2. **KanjiVG** - Stroke order animation (high educational value)
3. **KRADFILE** - Radical-based lookup
4. **Jisho API** - Fallback/online lookup

### License Compliance
- All sources require attribution
- KANJIDIC2: CC BY-SA 4.0
- JMdict/EDRG: Attribution required
- KanjiVG: CC BY-SA 3.0
- All allow commercial use with proper attribution

## Quick Start Code Example

```javascript
// Example: Loading KANJIDIC2 JSON in JavaScript
const kanjidicData = require('./kanjidic.json');

// Look up a kanji
function lookupKanji(char) {
  return kanjidicData.find(k => k.literal === char);
}

// Get readings
function getReadings(kanji) {
  const entry = lookupKanji(kanji);
  if (!entry) return null;
  
  return {
    on: entry.on_readings || [],
    kun: entry.kun_readings || [],
    meaning: entry.meanings?.en || []
  };
}

// Example: getReadings('漢')
// Returns: { on: ['カン'], kun: ['かん.じる'], meaning: ['China', 'Sino-'] }
```
